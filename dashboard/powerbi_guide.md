# Power BI Dashboard вҖ” Build Guide

## Overview

This guide walks you through building the North Star Workload Optimizer dashboard in Power BI Desktop (free). The dashboard consists of 4 pages with 5 KPI cards, time-trend charts, heatmaps, and anomaly detection views.

---

## Prerequisites

1. **Power BI Desktop** вҖ” [Download free](https://powerbi.microsoft.com/desktop/)
2. **Data file** вҖ” `data/expenses.csv` (generated by `generate_data.py`)
3. **DAX Reference** вҖ” See `dax_measures.md` for all measure formulas

---

## Step 1: Connect to Data

### Option A: CSV Import (Recommended вҖ” Most Portable)

1. Open Power BI Desktop
2. Click **Home вҶ’ Get Data вҶ’ Text/CSV**
3. Navigate to `data/expenses.csv`
4. In the preview dialog, verify:
   - Delimiter: Comma
   - Data Type Detection: Based on entire dataset
5. Click **Transform Data** to open Power Query Editor

### Option B: SQLite via ODBC

1. Install [SQLite ODBC Driver](http://www.ch-werner.de/sqliteodbc/)
2. **Get Data вҶ’ ODBC**
3. Connection string: `Driver={SQLite3 ODBC Driver};Database=<path_to>\northstar.db`
4. Select `fact_expenses` table

---

## Step 2: Transform Data in Power Query

In the Power Query Editor:

### 2.1 Set Column Types
```
transaction_date    вҶ’ Date
submission_date     вҶ’ Date
approval_date       вҶ’ Date
amount              вҶ’ Decimal Number
approval_lag_days   вҶ’ Whole Number
submission_lag_days вҶ’ Whole Number
confidence_score    вҶ’ Decimal Number
receipt_attached    вҶ’ True/False
policy_compliant    вҶ’ True/False
anomaly_flag        вҶ’ True/False
```

### 2.2 Add Calculated Columns
1. **Month Column**: Add Column вҶ’ Date вҶ’ Month вҶ’ Name of Month
2. **Year-Month**: Add Column вҶ’ Custom Column вҶ’ `Date.ToText([transaction_date], "yyyy-MM")`
3. **Quarter**: Add Column вҶ’ Date вҶ’ Quarter вҶ’ Quarter of Year

### 2.3 Click "Close & Apply"

---

## Step 3: Create Calendar Table

In the modeling view:

1. Click **Home вҶ’ New Table**
2. Enter the Calendar DAX formula from `dax_measures.md`
3. Create relationship: `Calendar[Date]` вҶ’ `fact_expenses[transaction_date]`

---

## Step 4: Create DAX Measures

Go to **Modeling вҶ’ New Measure** for each:

| # | Measure Name | Formula (see dax_measures.md) |
|---|-------------|-------------------------------|
| 1 | Total Expenses YTD | `CALCULATE(SUM(...), DATESYTD(...))` |
| 2 | Avg Approval Lag | `AVERAGE(fact_expenses[approval_lag_days])` |
| 3 | Anomaly Rate % | `DIVIDE(COUNTROWS(FILTER(...)), ...)` |
| 4 | Policy Compliance % | `DIVIDE(COUNTROWS(FILTER(...)), ...)` |
| 5 | MoM Growth % | `VAR CurrentMonth = ...` |

**KPI Cards Setup:**
1. Insert вҶ’ Card visual
2. Drag the measure to the "Fields" well
3. Format:
   - Font size: 28pt
   - Color: Use conditional formatting from `Approval Lag Color` measure
   - Add data labels with units ($, %, days)

**Monthly Trend Chart:**
1. Insert вҶ’ Line Chart
2. X-axis: `Calendar[YearMonth]`
3. Y-axis: Sum of `amount`
4. Legend: `department` (optional вҖ” for multi-line)
5. Format: Add trend line, markers

### Page 2: Department Deep Dive

**Department Heatmap:**
1. Insert вҶ’ Matrix visual
2. Rows: `department`
3. Columns: `category`
4. Values: Sum of `amount`
5. Format вҶ’ Background color вҶ’ Conditional formatting (gradient: green вҶ’ yellow вҶ’ red)

**Top Spenders Table:**
1. Insert вҶ’ Table visual
2. Columns: `employee_name`, `department`, Sum of `amount`
3. Sort by amount descending
4. Add `Employee Rank` measure
5. Format: Alternating row colors, data bars on amount

### Page 3: Anomaly Detection

**Anomaly Flag Table:**
1. Insert вҶ’ Table visual
2. Filter: `anomaly_flag = TRUE`
3. Columns: `expense_id`, `employee_name`, `department`, `amount`, `anomaly_type`, `confidence_score`
4. Conditional formatting on `confidence_score` (low = red)

**Amount vs Confidence Scatter:**
1. Insert вҶ’ Scatter Chart
2. X-axis: `amount`
3. Y-axis: `confidence_score`
4. Legend: `anomaly_type`
5. Size: fixed
6. Add reference lines at confidence = 0.5

### Page 4: Approval Pipeline

**Approval Lag Histogram:**
1. Insert вҶ’ Clustered Column Chart (or Histogram if available)
2. Axis: `approval_lag_days` (binned)
3. Values: Count of `expense_id`
4. Format: Add reference line at 3 days (target SLA)

**Speed Category Stacked Bar:**
1. Insert вҶ’ Stacked Bar Chart
2. Axis: `department`
3. Values: Count of `expense_id`
4. Legend: `approval_speed`
5. Colors: Fast=Green, Normal=Blue, Slow=Orange, Critical=Red

## Step 6: Add Slicers

Add the following slicers to each page (sync across pages):

1. **Department** вҖ” Dropdown slicer
2. **Date Range** вҖ” Date range slider using `Calendar[Date]`
3. **Category** вҖ” Checkbox slicer
4. **Status** вҖ” Dropdown slicer

To sync slicers across pages:
- View вҶ’ Sync slicers вҶ’ Check all pages

---

## Step 7: Apply Theme

1. View вҶ’ Themes вҶ’ Customize current theme
2. Recommended palette:
   - Primary: `#1B2A4A` (dark navy)
   - Secondary: `#2E86AB` (teal blue)
   - Accent 1: `#FFD23F` (gold/yellow вҖ” EY inspired)
   - Accent 2: `#EE6C4D` (coral)
   - Accent 3: `#2ECC71` (green)
   - Background: `#F8F9FA` (light gray)
   - Font: Segoe UI

## Step 8: Export

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Dates not recognized | Ensure `transaction_date` column type is set to Date in Power Query |
| Measures not working | Check that Calendar table relationship is active (1:Many) |
| Blank visuals | Verify data is loaded: Data view вҶ’ check row count |
| Slow performance | Reduce visual count per page, use Import mode |
